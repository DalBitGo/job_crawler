## 구현 방식 검토

| 방식 | 장점 | 단점 | 적합도 |
|------|------|------|--------|
| **Notion DB** | 빠른 구축, 모바일 접근, 칸반 뷰 | 크롤러 연동 복잡 (API 필요) | ⭐⭐⭐ |
| **Streamlit 웹앱** | Python 기반, 자동화 쉬움, 자유도 높음 | 직접 개발, 호스팅 필요 | ⭐⭐⭐⭐ |
| **Google Sheets** | 단순함, 공유 쉬움 | 기능 제한, UI 불편 | ⭐⭐ |
| **CLI + SQLite** | 가볍고 빠름 | 시각화 부족 | ⭐⭐ |

## 데이터 구조 (안)

```
jobs (공고 테이블)
├── id (PK)
├── source (사람인/잡코리아/원티드)
├── company
├── title
├── link
├── tech_stack (JSON)
├── conditions (JSON)
├── crawled_at
├── status (new/bookmarked/applied/passed/interview/rejected/offer)
├── applied_at
├── notes
└── created_at

application_history (상태 변경 이력)
├── id (PK)
├── job_id (FK)
├── from_status
├── to_status
├── changed_at
└── memo
```
