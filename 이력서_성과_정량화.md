## 핵심 성과 (암기할 것)

### 1. 파이프라인 규모
```
"22개 고객사의 데이터 파이프라인을 Airflow로 운영"
```
- 22개 고객사 DAG
- 일일 수백만 건 데이터 처리
- 복잡한 의존성 관리

### 2. 아키텍처 개선 (Unified Converter)
```
"3개 서비스를 1개로 통합하여 성능 60% 개선"
```

| 지표 | Before | After | 개선율 |
|------|--------|-------|--------|
| Cloud Function 수 | 3개 | 1개 Cloud Run | 67%↓ |
| API 호출 | 3회 | 1회 | 67%↓ |
| Airflow Task | 6-8개 | 2-3개 | 75%↓ |
| 처리 시간 | ~5분 | ~2분 | **60%↓** |
| 장애 포인트 | 3개 | 1개 | 67%↓ |

### 3. 자동화 서비스
```
"80개 이상의 Python 자동화 서비스 개발/운영"
```
- 80+ 마이크로서비스
- 일일 수작업 6시간 절감

### 4. 모니터링 시스템
```
"4개의 모니터링 시스템을 직접 개발하여 장애 대응 시간 50% 단축"
```
- airflow_dag_monitor
- history_checker
- converter_failure_monitor
- sli_slo

## 면접 답변용 문장

### "Airflow 경험을 설명해주세요"
> "22개 고객사의 데이터 파이프라인을 Airflow로 설계하고 운영했습니다.
> DAG 간 복잡한 의존성 관리, 스케줄링 최적화, 백필 처리를 담당했고,
> DAG 모니터링 시스템을 직접 개발하여 장애 대응 시간을 50% 단축했습니다."

### "대용량 데이터 처리 경험이 있나요?"
> "일일 수백만 건의 데이터를 BigQuery와 Airflow로 처리하고 있습니다.
> 파이프라인 최적화를 통해 처리 시간을 60% 단축한 경험이 있으며,
> 현재 개인 프로젝트에서 Kafka + Spark로 더 큰 규모의 처리를 학습하고 있습니다."

### "성능 개선 경험을 말해주세요"
> "Unified Converter 프로젝트에서 3개의 분산된 Cloud Function을
> 1개의 Cloud Run으로 통합했습니다.
> 그 결과 처리 시간 60% 단축, API 호출 67% 감소, 장애 포인트 67% 감소를 달성했습니다."

### "Spark 경험이 부족한데요?"
> "실무에서는 BigQuery와 Airflow로 대용량 데이터를 처리했습니다.
> 분산 처리의 핵심 개념인 파티셔닝, 셔플 최소화는 이해하고 있으며,
> 현재 개인 프로젝트에서 Kafka + Spark Streaming을 구현 중입니다.
> 초당 1만 건 처리를 목표로 하고 있습니다."
